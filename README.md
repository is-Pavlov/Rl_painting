# Гибридный промышленный оптимизатор для автоматизации колеровки ЛКМ (DQN + L-BFGS-B)

Система автоматического подбора рецептуры лакокрасочных материалов на базе глубокого обучения с подкреплением и классических градиентных методов оптимизации

## Аннотация
*Работа посвящена автоматизации доведения цвета при производстве лакокрасочных материалов (ЛКМ).Традиционные методы зависимы от человеческого фактора, что затягивает производственный цикл*

*В проекте предложен подход на базе обучения с подкреплением (Reinforcement Learning, RL), моделирующий подбор рецептуры как Марковский процесс принятия решений (Markov decision process, MDP). Разработана цифровая среда смешивания, учитывающая нелинейное взаимодействие пигментов. Агент Deep Q-Network (DQN) в автоматическом режиме определяет дозировку паст для минимизации цветовых отклонени. Сравнительный анализ подтвердил превосходство RL-подхода при соблюдении технологических лимитов

---

## Ключевые особенности проекта
**3-слойный физико-химический симулятор:** Имитирует нелинейный процесс смешивания 11 пигментных паст. Модель откалбирована по реальным производственным данным методом наименьших квадратов (слой Linear Scale), учитывает нелинейный эффект насыщения среды через гиперболический тангенс (слой Nonlinear Scaling) и моделирует перекрестное влияние пигментов (слой Cross-interaction).
**Механизм PER (Prioritized Experience Replay):** Решает проблему малых и ограниченных данных на производстве. Алгоритм приоритизирует шаги с максимальной ошибкой временного различия (TD-error), что позволяет извлекать максимум знаний из небольшого объема накопленного опыта.
**Гибридный оптимизатор:** Сочетает DQN-агента и классический алгоритм L-BFGS-B. Нейросеть выступает в роли глобального стратега, определяя основу рецепта и защищая систему от застревания в локальных минимумах. Градиентный метод L-BFGS-B проводит локальный спуск для прецизионного расчета граммовок, строго соблюдая ограничения по массе

## Результаты внедрения
**Сокращение времени:** Время контроля качества и подбора рецептуры сокращено в 5–6 раз. Традиционный ручной подбор требовал 3–4 итерации (около 120 минут). С разработанной моделью требуется всего 1 итерация (20–25 минут), средняя скорость работы самого алгоритма генерации рецепта составляет **4.2 секунды**.
**Точность попадания:** Обеспечено снижение стартового цветового отклонения $\Delta E$ с 9.0 до **1.8 единиц**
**Контроль лимитов:** Гарантировано соблюдение жестких технологических ограничений по массе вводимых паст (не более 3%), что полностью исключает риск перепигментации
